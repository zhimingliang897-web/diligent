## 一、服务器Nginx配置优化

### 1.1 问题背景

服务器 `8.138.164.133` 原本配置了 HTTPS 强制跳转，但由于证书问题导致浏览器缓存了 HSTS，无法正常访问。

### 1.2 解决方案

修改 `/etc/nginx/conf.d/api.liangyiren.top.conf`，移除 HTTPS 强制跳转，改为同时支持 IP 和域名访问。

**修改前：**
```nginx
# HTTP -> HTTPS
server {
    listen 80;
    server_name api.liangyiren.top;
    return 301 https://$server_name$request_uri;
}
```

**修改后：**
```nginx
# HTTP服务 - 同时支持IP和域名访问
server {
    listen 80;
    server_name api.liangyiren.top 8.138.164.133;
    client_max_body_size 100m;
    # ... 反向代理配置
}
```

### 1.3 浏览器HSTS清理方法

如果浏览器仍然跳转 HTTPS，需要清除 HSTS 缓存：

1. Chrome 地址栏输入 `chrome://net-internals/#hsts`
2. 在 "Delete domain security policies" 输入 `api.liangyiren.top`
3. 点击 Delete，重启浏览器
