# 苍穹外卖 (Sky Takeout) - 4天极速通关与面试突击计划（Day 0 已完成）

## **项目简介与冲刺策略**
“苍穹外卖”是一个经典的 Java 企业级实战项目，技术栈覆盖 Spring Boot, MyBatis Plus, Redis, MySQL, 微信小程序等。
鉴于 **Day 0 (环境配置、数据库初始化、Maven构建)** 已经顺利完成，本计划专为剩下的 **核心业务** 打造 **4天极速冲刺**。

**核心策略：**
1.  **抓大放小**：针对面试最高频的考点（登录鉴权、AOP填充、跨表事务、Redis缓存、各种锁及超时订单）实战落地。
2.  **拒绝搬砖**：跳过简单的 CRUD 机械重复工作，专注于复杂业务链路（如：包含口味的多表菜品管理、微信支付与下单秒杀）。

## **Day 01 - 基础防护墙：用户鉴权与全局配置**

### **今日目标**
彻底跑通员工的登录请求与基础信息管理，搭建项目必备的“防线”。

### **详细任务拆解**
1.  **JWT 登录与拦截**：
    -   实现 `/admin/employee/login`。
    -   配置 `JwtTokenAdminInterceptor` 拦截器，解析 Token，将 UserId 存入 `ThreadLocal`。
2.  **员工管理核心 CRUD**：
    -   **新增员工**: 捕获 `SQLIntegrityConstraintViolationException` (账号重复)，处理 MD5(123456) 默认密码。
    -   **分页查询**: 使用 `PageHelper`。
    -   **状态流转**: 启用/禁用员工账号（`Builder` 模式更新实体）。
3.  **全局基础组件整合**：
    -   `Result<T>` 泛型统一响应结果。
    -   `GlobalExceptionHandler` 全局业务与 SQL 异常捕获机制。
    -   Swagger / Knife4j 接口文档自动生成。

### **面试 CheckList**
- [ ] ThreadLocal 在本项目的工作流是什么？为什么请求结束必须 `remove()`？
- [ ] JWT 的组成是哪三部分？相比传统 Session 登录有何优劣？

## **Day 02 - 后台纵深：AOP 切面与多表事务**

### **今日目标**
利用进阶技术消除模板代码（公共字段），并处理具有多表关联关系的菜品复杂业务。

### **详细任务拆解**
1.  **AOP 公共字段填充（面试重点）**：
    -   自定义注解 `@AutoFill`，标记 `OperationType.UPDATE` / `INSERT`。
    -   编写 `AutoFillAspect` 切面，拦截 DAO 层 Mapper 执行前，利用 Java 反射动态写入 `create_time`, `update_user` 等字段。
2.  **对象存储与文件上传**：
    -   实现菜品图片的阿里云 OSS 上传（或本地相对路径静态资源映射）。
3.  **菜品（Dish）与口味管理**：
    -   新增菜品时，包含动态口味列表，需开启 `@Transactional` 保证 `dish` 和 `dish_flavor` 这两张表同生共死。
    -   改造分类管理的普通 CRUD。

### **面试 CheckList**
- [ ] Spring AOP 的底层实现是 JDK 动态代理还是 CGLIB？怎么选？
- [ ] `@Transactional` 在什么情况下会失效？

## **Day 03 - 移动端破局：小程序登录与 Redis 提速**

### **今日目标**
从 B 端转向 C 端，集成微信登录体系，利用 Redis 为菜品查询接口扛并发。

### **详细任务拆解**
1.  **微信体系免密登录**：
    -   基于 `HttpClient` 调用微信接口服务，通过小程序端传来的 `code` 换取 `OpenID`。
    -   首次登录自动完成用户静默注册，颁发 C 端 JWT。
2.  **Redis 性能大考**：
    -   使用 Spring Data Redis 或 Spring Cache 缓存 C 端用户频繁访问的菜品列表数据。
    -   难点清理：商家在后台编辑分类、菜品及状态变更时，**精准淘汰**影响到的 Redis 缓存（缓存数据一致性）。
3.  **购物车与会话暂存**：
    -   基于 MySQL `shopping_cart` 或替代方案（Redis的 Hash 结构）存取购物车条目。

### **面试 CheckList**
- [ ] 更新数据库时，是先更新数据库还是先删缓存？为什么本项目选这种方案？
- [ ] 微信小程序的登录交互图能画出来吗？

## **Day 04 - 闭环之战：订单状态机与定时/实时推送**

### **今日目标**
攻克外卖系统中最复杂的“订单流转”，并加入实用的定时任务和全双工长链接交互。

### **详细任务拆解**
1.  **生成用户订单**：
    -   前端提交，后端参数校验，从购物车拉取数据，锁定商品并插入订单及明细表（需事务包裹）。
    -   清空购物车数据。
2.  **微信支付（沙箱或模拟）**：
    -   了解微信支付的 `prepay_id` 获取流程。
    -   处理支付成功后的异步通知回调，驱动订单状态从 `PENDING_PAY` -> `TO_BE_ACCEPTED`。
3.  **超时未支付处理**：
    -   使用 **Spring Task** 设置定时任务（如每 1 分钟扫表），对超时达 15 分钟的订单自动取消关单，释放库存。
    -   *(进阶方向：可探讨 RabbitMQ 延迟队列替代方案）*
4.  **商家抢单与 WebSocket 播报**：
    -   基于 WebSocket 建立后端与商家网页管理端的全双工长链接。
    -   用户下单支付成功后，后端主动向商家端推送 JSON 消息，触发前端语音播报机制。

### **面试 CheckList**
- [ ] 定时任务轮询数据库有什么缺点？大规模订单该如何优化？
- [ ] WebSocket 解决了 HTTP 的什么痛点？
